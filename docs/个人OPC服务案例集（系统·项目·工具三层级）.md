## 一、我是谁，我能帮你解决什么问题

> **"企业的业务里，有些环节是人在做机器该做的事。我帮你把那些环节交给 AI，让你的团队去做只有人才能做的事。"**

我是一个**全链路数智化交付者**。不是咨询顾问——我不交 PPT。不是外包程序员——我不接"照着做"的活。我的工作方式是：理解业务逻辑，找到自动化切入点，设计技术方案，交付可运行的系统。

我的竞争力来自三件事同时具备：

```
教学深度（能把复杂技术讲明白，让团队能接手）
    ×
工程落地（能把方案真正做出来，不只停留在文档）
    ×
科研前瞻（能把前沿技术转化为可落地的业务方案）
    =
单一角色无法替代的交付能力
```

以下 30 个案例按**系统层级 → 项目层级 → 工具层级**三层展开，每一层对应不同的客户需求深度和交付复杂度。

## 三、系统层级 —— 把 AI 能力嵌入业务架构

系统层级的核心不是"做了一个软件装在你电脑上"——是**深度参与业务架构设计，从数据架构到流程再到人机协作模式全面重构**。

---

### 3.1 林翼巡无人机巡检监管系统

| 维度 | 详情 |
|------|------|
| **适用场景** | 林业管理部门需要对大片林区进行常态化巡检，人工巡检效率低、覆盖面小 |
| **业务架构变革** | 原模式：人工巡查→纸质记录→电话上报→层层审批。新模式：大疆机场2自动起飞→AI实时识别→系统自动告警→一键生成报告 |
| **技术架构** | YOLOv8 目标检测（烟火/车辆/违规行为）+ ByteTrack 多目标跟踪 + Qwen3-VL VLM 场景理解 + Jetson Orin Nano 边缘推理 |
| **架构设计** | 边缘端离线运行（林区无信号可工作），数据回传后云端同步；大疆机场2自动换电/充电，支持 7×24 无人值守 |
| **交付规模** | 9841 个文件，11 份 GB/T 标准工程文档 |

**业务价值**：
- 巡查半径从每人每天 5 公里 → 单架次覆盖 50 公里
- 异常事件从"事后发现" → 实时告警
- 巡检报告从人工 2 小时 → 系统自动生成 5 分钟

|------|------|

## 四、项目层级 —— 让业务完成一次数智化升级

项目层级解决的是：客户的某个业务环节现有模式已经满足不了发展需要。交付的是**把业务逻辑翻译成技术路径的方案**，让业务完成一次明确的升级。

---

### 4.1 AI 项目服务链六环节全方案

| 维度 | 详情 |
|------|------|
| **业务现状** | 企业想落地 AI 但不知道从哪入手，或做了单环节（如买了标注工具）但形成不了闭环 |
| **升级路径** | 从"散点试错"→"全链路可控"：数据集获取→智能标注→数据增强→模型训练→部署测试→工程化，六个环节的通盘方案 |
| **方案架构** | 每个环节独立方案文档（6 份 GB/T 标准）+ 全链路集成方案 + ROI 测算 |
| **交付特点** | 甲方视角撰写——不堆砌术语，讲清楚"这么做能得到什么、不这么做会踩什么坑" |

**典型案例**：
- **烟田地块分割**：YOLO 实例分割 + 月度随机森林 + 软投票集成，含数据源选择与样本标注策略
- **光伏板缺陷检测**：EL 影像 + 可见光 + 红外三通道并行，含数据增强策略和边缘端部署方案
- **林火烟雾检测**：多尺度特征融合 + 时序上下文建模，含误报控制策略

**业务价值**：不是买一堆工具——是建立"数据→模型→系统"的可复制能力。烟田方案做完后，方法论可迁移到其他作物识别。
